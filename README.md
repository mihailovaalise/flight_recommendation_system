# flight_recommendation_system
Выпускная квалификационная работа: **"Разработка рекомендательной системы на основе графовых нейронных сетей"**.

Проект посвящен построению рекомендательной системы для персонализированного подбора авиамаршрутов и авиарейсов на основе данных о бронированиях пассажиров.  
В системе используется графовое представление взаимодействий пользователей и перелетов.

Система реализует двухэтапный подход:
1. на первом этапе пользователю рекомендуются наиболее релевантные маршруты;
2. на втором этапе внутри выбранного маршрута подбираются наиболее подходящие конкретные рейсы.

Проект также включает веб-интерфейс для демонстрации работы рекомендательной системы.

## Состав проекта

- `data_processing.py` — скрипт предобработки и подготовки данных.  
  В этом файле выполняется загрузка и очистка исходных таблиц, объединение данных о пассажирах, рейсах и бронированиях, формирование признаков, кодирование категориальных переменных, построение пользовательско-маршрутного графа и подготовка датасета для обучения моделей.

- `models.py` — скрипт построения и оценки моделей рекомендаций.  

- `site/` — папка с веб-приложением для демонстрации системы рекомендаций.  

## Используемые данные

В проекте используются данные о пассажирах, рейсах и бронированиях, на основе которых формируется граф взаимодействий между пользователями и перелетами.  
Ребра графа отражают факты бронирования, а также позволяют учитывать пользовательскую историю перелетов, популярность направлений и структуру маршрутной сети.

В процессе подготовки данных могут использоваться такие признаки, как:
- идентификатор пассажира;
- идентификатор рейса или маршрута;
- аэропорт вылета;
- аэропорт прилета;
- дата бронирования;
- дата вылета;
- класс бронирования;
- статус бронирования;
- количество предыдущих бронирований пользователя;
- число уникальных маршрутов пользователя.

## Используемые модели

В проекте рассматриваются и сравниваются следующие модели и подходы:

- **baseline-модели** — базовые табличные подходы, используемые для сравнения с графовыми архитектурами;
- **LightGCN** — основная графовая модель рекомендаций, использующая распространение эмбеддингов по графу взаимодействий пользователей и маршрутов;
- **GCN** — графовая сверточная сеть, учитывающая структуру графа и, при необходимости, дополнительные признаки узлов;
- **GraphSAGE** — модель, основанная на агрегации признаков соседних узлов и также рассматриваемая в варианте с дополнительными признаками;
- **MLP re-ranker** — модель для доранжирования маршрутов, применяемая поверх графовой модели после отбора кандидатов;
- **ранкер рейсов внутри маршрута** — отдельный этап, используемый для выбора наиболее подходящих конкретных авиарейсов внутри рекомендованного маршрута.

По результатам экспериментов для формирования итоговых персонализированных рекомендаций в веб-интерфейсе используется двухэтапная схема, в которой графовая модель формирует список наиболее релевантных маршрутов, а дополнительные модели уточняют итоговый порядок маршрутов и конкретных рейсов.

## Оценка качества рекомендаций

Для оценки качества рекомендательной системы используются стандартные метрики ранжирования:

- `HitRate@k`
- `NDCG@k`
- `MRR`
- `Coverage@10`

Эти метрики позволяют оценить, насколько часто целевой маршрут попадает в список рекомендаций, насколько высоко он расположен в ранжированном списке и насколько разнообразными являются выдаваемые рекомендации.

## Как запустить

1. Установите необходимые библиотеки:
```bash
pip install pandas numpy scikit-learn torch torch-geometric streamlit
```

2. Подготовьте данные и разместите их в рабочей директории проекта.

3. Запустите предварительную обработку данных:
```bash
python data_processing.py
```

4. Запустите обучение и оценку моделей:
```bash
python models.py
```

5. После подготовки необходимых файлов для инференса откройте папку `site/` и запустите веб-приложение в соответствии с используемой реализацией интерфейса.
